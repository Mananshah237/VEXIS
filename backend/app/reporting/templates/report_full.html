Glossary

TermDefinition
Taint analysisStatic analysis technique that tracks untrusted input (sources) through a program to dangerous operations (sinks)
SourceA location where untrusted user-controlled data enters the program (e.g., request.args.get)
SinkA location where data is used in a security-sensitive operation (e.g., db.execute, os.system)
SanitizerCode that transforms untrusted data into a safe form (e.g., parameterized queries, html.escape)
Cross-file taintTaint flows that cross function call boundaries or module imports
Attack chainMultiple individually lower-severity vulnerabilities that combine into a higher-severity exploit path
Second-order injectionAttack where malicious data is stored in a database and triggered in a later request
TOCTOUTime-Of-Check-Time-Of-Use race condition between a security check and the protected operation

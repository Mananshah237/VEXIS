# VEXIS Launch Checklist

Pre-launch verification checklist for the VEXIS v0.1.0 public release.

---

## Infrastructure

- [ ] PostgreSQL provisioned (Railway / Fly / self-hosted)
- [ ] Redis provisioned
- [ ] MinIO / R2 / S3 bucket created and credentials set
- [ ] Backend deployed and `/health` returns `{"status":"ok"}`
- [ ] Frontend deployed and loads without console errors
- [ ] `NEXT_PUBLIC_API_URL` and `NEXT_PUBLIC_WS_URL` match the deployed backend
- [ ] Database migrations applied (`alembic upgrade head`)

## Auth

- [ ] GitHub OAuth App created at github.com/settings/developers
- [ ] Callback URL set to `https://<backend>/api/v1/auth/callback/github`
- [ ] `GITHUB_CLIENT_ID` and `GITHUB_CLIENT_SECRET` set in backend env
- [ ] `JWT_SECRET` set to a strong random value (≥32 chars)
- [ ] `NEXTAUTH_SECRET` set in frontend env
- [ ] `NEXTAUTH_URL` set to frontend origin
- [ ] Sign-in flow works end-to-end (GitHub → token → dashboard)

## Security

- [ ] `CORS_ORIGINS` restricted to frontend origin (not `*`)
- [ ] `JWT_SECRET` is not the default `dev-secret-change-in-production`
- [ ] `MINIO_SECRET_KEY` is not the default `vexis_dev_password`
- [ ] Rate limiting verified: 4th scan in 24h returns HTTP 429
- [ ] `/api/v1/scan/{id}` returns 404 for cross-user access attempts

## Core scan functionality

- [ ] Raw code scan completes end-to-end (submit → poll → findings)
- [ ] GitHub URL scan clones repo and returns findings
- [ ] WebSocket progress updates reach the browser during a scan
- [ ] Taint path displayed correctly on finding detail page
- [ ] PoC / AI reasoning displayed on finding detail page
- [ ] Triage (mark true positive / false positive) persists

## PDF reports

- [ ] WeasyPrint system deps installed in Docker image (`libpango`, `libgdk-pixbuf`, `libcairo2`)
- [ ] `GET /api/v1/report/{scan_id}` returns a valid PDF for a completed scan
- [ ] "Download PDF Report" button visible and functional on scan results page
- [ ] PDF cover page shows correct scan metadata and risk score
- [ ] Finding detail pages render correctly in the PDF

## Incremental scanning

- [ ] First scan of a source saves manifest to MinIO (`scan-artifacts/{id}/manifest.json`)
- [ ] Second scan with `incremental: true` skips unchanged files
- [ ] "Re-scan (incremental)" button on scan results page triggers a new scan
- [ ] `GET /api/v1/scan/{id}/compare/{other_id}` returns new/resolved/unchanged lists

## GitHub Action

- [ ] `action/action.yml` references correct Docker image
- [ ] Example workflow in `action/README.md` is copy-pasteable and correct
- [ ] Action fails with exit 1 when findings exceed severity threshold
- [ ] GitHub PR annotations appear for findings (test in a real repo)

## Documentation

- [ ] `README.md` accurately describes current feature set
- [ ] `DEPLOY.md` covers Railway, Fly.io, storage options, and all env vars
- [ ] `ARCHITECTURE.md` reflects Sprint 6 + 7 additions
- [ ] `action/README.md` has correct input/output table
- [ ] `LICENSE` file present (MIT)

## Final smoke test

- [ ] End-to-end scan of a known-vulnerable repo (e.g. flask SQLi example) returns ≥1 critical finding
- [ ] PDF report generated for that scan opens correctly in a PDF viewer
- [ ] GitHub Action fires on a test PR and annotates the vulnerable lines
- [ ] Dashboard shows scan history with pagination working at >15 scans
